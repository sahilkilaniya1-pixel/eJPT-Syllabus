🛠️ DETAILED EXPLANATION OF SYMBOLS & MECHANICS (Hinglish Notes)
----------------------------------------------------------------------------------------------------

1. MEANING OF TERMINOLOGIES:
   - D (Data): Original user message ya application data.
   - H (Header): Control information jo har layer attach karti hai (Addresses, Sequence numbers, etc.).
   - T2 (Trailer): Data Link Layer (Layer 2) ke end par lagne wala Error Checking code (FCS/CRC).

2. SENDER SIDE: DATA ENCAPSULATION (Top-to-Bottom Flow ⬇️)
   - Application Layer (L7) user data generate karta hai aur apna Header (H7) lagata hai.
   - Presentation (L6) aur Session (L5) layers respective headers (H6, H5) add karti hain.
   - Transport Layer (L4) header (H4) lagati hai (Port numbers & Sequence numbers).
   - Network Layer (L3) header (H3) add karti hai (Logical IP Addresses).
   - Data Link Layer (L2) header (H2 - MAC Address) AND TRAILER (T2 - Error Detection) dono attach karti hai.
   - Physical Layer (L1) poore packet aur headers ko Raw Binary Bits (0s & 1s) aur Signals mein badal deti hai.

   > KEY CONCEPT: Har pichli layer ka poora message (Data + Header) agli layer ke liye sirf 'Data (D)' ban jata hai.

3. PHYSICAL TRANSMISSION MEDIUM:
   - Encapsulated bit-stream coaxial, twisted-pair, fiber-optic cable, ya wireless signals ke zariye 
     destination device tak travel karti hai.

4. RECEIVER SIDE: DATA DECAPSULATION (Bottom-to-Top Flow ⬆️)
   - Layer 1 physical signals ko binary bits (0s & 1s) mein convert karti hai.
   - Layer 2 Trailer (T2) se error check karti hai aur H2 header read karke peel-off (hata) deti hai.
   - Layer 3 se Layer 7 tak har layer apna-apna respective Header (H3 ➔ H4 ➔ H5 ➔ H6 ➔ H7) read karti hai, 
     use process karti hai aur strip (hata) deti hai.
   - Aakhir mein original clean message (Data) user screen par render ho jata hai.

----------------------------------------------------------------------------------------------------
1. INTRODUCTION TO TCP/IP MODEL
----------------------------------------------------------------------------------------------------
TCP/IP (Transmission Control Protocol / Internet Protocol) woh architectural framework hai 
jis par aaj ka poora Global Internet aur Web applications run karte hain.

🛠️ KEY HIGHLIGHTS & HISTORY:
- OSI Model Practical vs Theoretical: OSI reference model ek theoretical model tha, jabki TCP/IP 
  ek practical model hai jo reality mein internet standard ban gaya.
- OSI vs TCP/IP Conciseness: TCP/IP OSI model ka ek simplified version hai. Isliye iski layers 
  OSI model se exact match nahi karti.
- Layer Evolution: 
  * Original Model: Pehle TCP/IP ko Hardware ke upar 4 Software Layers ka model mana jata tha.
  * Modern Model: Aaj ke time par TCP/IP ko 5-Layer Model ke roop mein padha aur use kiya jata hai.

----------------------------------------------------------------------------------------------------
3. 5-LAYER TCP/IP MODEL DEEP DIVE & PROTOCOLS
----------------------------------------------------------------------------------------------------

:: LAYER 5: APPLICATION LAYER ::
- OSI ke Application, Presentation, aur Session teeno layers ke kaam (UI, formatting, encryption, 
  aur session control) akele yahi layer handle karti hai.
- Common Protocols: HTTP, HTTPS, FTP, SMTP, DNS, SSH, Telnet, DHCP.

:: LAYER 4: TRANSPORT LAYER ::
- End-to-End communication, error control, aur process-to-process delivery handle karti hai.
- Common Protocols:
  * TCP (Transmission Control Protocol): Connection-oriented & reliable (e.g., Web browsing, Email).
  * UDP (User Datagram Protocol): Connection-less & super fast (e.g., Video calls, Gaming, Streaming).

:: LAYER 3: NETWORK / INTERNET LAYER ::
- Packets ka logical address (IP) handle karti hai aur shortest path routing karti hai.
- Common Protocols: IP (IPv4 / IPv6), ICMP (Ping), ARP (Address Resolution), IPsec.

:: LAYER 2: DATA LINK LAYER ::
- Local network/node-to-node framing aur hardware MAC addressing handle karti hai.
- Common Standards: Ethernet (IEEE 802.3), Wi-Fi (IEEE 802.11), PPP.

:: LAYER 1: PHYSICAL LAYER ::
- Raw binary bits (0s & 1s) ko physical signals (copper electric pulses, light waves, radio waves) 
  mein badal kar transmit karti hai.
