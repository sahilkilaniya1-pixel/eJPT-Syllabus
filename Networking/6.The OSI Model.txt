----------------------------------------------------------------------------------------------------
1. OPEN SYSTEMS & COMMUNICATION ARCHITECTURE
----------------------------------------------------------------------------------------------------
ISO (International Organization for Standardization) ne alag-alag systems ke beech seamless 
communication ke liye OSI Model ko architect kiya.

🛠️ SYSTEM TYPES:
- Isolated / Closed System: Wo system jo kisi doosre network ya device se communicate nahi kar sakta.
- Open System: Wo system jo standard rules (Protocols) ko follow karke kisi bhi doosre open system 
  se network par jud sakta hai.

🛠️ COMMUNICATION ARCHITECTURE (Kyun Zaroori Hai?):
- Strategy & Standards: Yeh hosts aur devices ke beech data communication ke zaroori elements 
  aur standards ko define karta hai.
- Modular Layering Flexibility: Modularity ka sabse bada fayda yeh hai ki agar hume koi naya hardware 
  ya network device add karna ho, toh hume poore host software ko rewrite nahi karna padta—sirf 
  relevat layer update hoti hai.

----------------------------------------------------------------------------------------------------
2. THE OSI REFERENCE MODEL OVERVIEW (7 Layers)
----------------------------------------------------------------------------------------------------
OSI Model computer communication ke complex process ko **7 CHHOTE AUR MANAGABLE STAGES (LAYERS)** 
mein todta hai.

📊 THE 7 LAYERS STACK (Top to Bottom):

+-------+-------------------+-----------------------------------+-----------------------------------+
| Layer | Layer Name        | Primary Function                  | Data Unit / Focus                 |
+-------+-------------------+-----------------------------------+-----------------------------------+
|   7   | Application Layer | Network Processes to Application  | User Interface & Services         |
|   6   | Presentation Layer| Data Representation & Encryption  | Translation & Formatting          |
|   5   | Session Layer     | Interhost Communication           | Dialog Management & Sync          |
|   4   | Transport Layer   | End-to-End Connections            | Process-to-Process (Segments)     |
|   3   | Network Layer     | Logical Addressing & Routing      | Path Determination (Packets)      |
|   2   | Data Link Layer   | Physical Addressing & Media Access| Hop-to-Hop Framing (Frames)       |
|   1   | Physical Layer    | Binary Transmission               | Raw Bits over Physical Medium     |
+-------+-------------------+-----------------------------------+-----------------------------------+

----------------------------------------------------------------------------------------------------
3. DEEP DIVE: LAYER-BY-LAYER BREAKDOWN
----------------------------------------------------------------------------------------------------

:: LAYER 7: APPLICATION LAYER (User Interface) ::
- Top-most layer jo user ke software applications (Browser, Email Clients, Database Apps) ko 
  direct network services provide karti hai.
- Responsibilities: End-user services jaise File Transfer (FTP), Electronic Mail (SMTP), 
  Web Browsing (HTTP/HTTPS), aur Remote Login (SSH/Telnet) handle karna.

:: LAYER 6: PRESENTATION LAYER (The Translator) ::
- Network communication ke liye data format aur syntax ko manage karta hai.
- Outgoing Data: Application data ko generic network format mein convert karta hai.
- Incoming Data: Generic network format se application-understandable format mein translate karta hai.
- Responsibilities: Protocol Conversions, Data Encryption / Decryption, Data Compression / Decompression.
- Redirector Feature: Ek special software facility ("Redirector") yahan decide karti hai ki request 
  local file ke liye hai ya network resource ke liye.

:: LAYER 5: SESSION LAYER (Dialog Controller) ::
- Do networked resources ke beech ongoing communication channel (Dialog Session) banata hai.
- Responsibilities:
  1. Session initiation, maintenance, aur termination handle karna.
  2. Security & Access Control (Participant identification).
  3. Synchronization Services (Session checkpoints lagana taaki connection drop hone par wahan se resync ho sake).

:: LAYER 4: TRANSPORT LAYER (End-to-End Delivery) ::
- Network par message ki process-to-process reliable transmission aur flow control manage karta hai.
- Responsibilities:
  1. Segmentation: Bade data streams ko chhote manageable chunks (Segments) mein todna.
  2. Reassembly: Receiving end par chunks ko unke original sequence mein dobara jodna.
  3. Error Control & Acknowledgement: Successful delivery ki acknowledgement dena aur lost data re-request karna.

:: LAYER 3: NETWORK LAYER (Logical Address & Routing) ::
- Logical Addressing (IP Addresses) handle karta hai aur data transfer ke liye BEST PATH (Shortest Route) chunat hai.
- Responsibilities:
  1. Routing decisions aur Logical-to-Physical address translation.
  2. Packet Switching aur Network Congestion Control handle karna.

:: LAYER 2: DATA LINK LAYER (Hop-to-Hop / Framing) ::
- Network Layer aur Physical Layer ke beech raw data ko structured DATA FRAMES mein convert karta hai.
- Responsibilities:
  1. Sending End: Data ko raw format mein convert karke Physical layer ko dena.
  2. Receiving End: Physical layer se aaye raw binary bits ko package karke structured Frames banana.
  3. Physical Addressing: MAC Address (Hardware Address) ka use karke local hop-to-hop transfer handle karna.

:: LAYER 1: PHYSICAL LAYER (Hardware & Bits) ::
- Bottom-most layer jo network medium (Coax, Twisted-Pair, Fiber) par individual BITS ko physically move karti hai.
- Responsibilities:
  1. Outgoing Message: Bits (0s & 1s) ko Electrical / Light signals mein badalna.
  2. Incoming Message: Signals ko wapas raw bits mein translate karna.
  3. Hardware Driver Interface: Network Interface Cards (NICs), MAU, aur Modems ke hardware drivers ko instruct karna.
