----------------------------------------------------------------------------------------------------
1. INTRODUCTION TO SWITCHING
----------------------------------------------------------------------------------------------------
Network mein hazaron devices hote hain. Unhe aapas mein connect karne ke do tarike ho sakte hain:

1. Direct Point-to-Point Connections: 
   Har device ko har doosre device se direct wire se jod do (Mesh). Lekin devices badhte hi 
   cables aur connections ki ginti asman choone lagti hai.
2. Switched Network (Central/Star Topology):
   Devices ko ek central device (Switch/Router) se connect kiya jata hai. Interconnected 
   switches ke network ko hi 'Switched Network' kehte hain.

----------------------------------------------------------------------------------------------------
2. MESSAGE SWITCHING (Store and Forward Mechanism)
----------------------------------------------------------------------------------------------------
Message switching mein poore data (message) ko ek single unit ki tarah mana jata hai jiske sath 
destination aur source ka address juda hota hai.

🛠️ WORKING MECHANISM:
- Intermediate devices (Switches) poore message ko pehle receive karti hain, use apne pass 
  STORE karti hain, aur jab agli device free hoti hai tab use FORWARD karti hain.
- Isiliye ise **"Store and Forward Switching"** bhi kaha jata hai.

🛠️ PROS & CONS OF MESSAGE SWITCHING:
* Advantages (Fayde):
  - Shared Channel: Ek hi channel ko bohot sare devices share kar sakte hain.
  - Reduced Congestion: Message tabhi aage bhejte hain jab agli line khali ho, jisse jam nahi lagta.
  - Unlimited Size: Kisi bhi size ka bada message bheja ja sakta hai.
* Disadvantages (Nuksan):
  - Not Compatible for Voice/Video: Delay zyada hone ke karan real-time calling/video ke liye bilkul bekaar hai.
  - High Memory Cost: Intermediate switches ko bada message store karne ke liye mehngi memory chahiye.
  - Security & Reliability: Intermediate nodes par data hack hone ka risk rehta hai.

----------------------------------------------------------------------------------------------------
3. CIRCUIT SWITCHING (Dedicated Path Mechanism)
----------------------------------------------------------------------------------------------------
Circuit switching ka use public telephone networks mein data aur voice calls handle karne ke 
liye shuru hua tha.

🛠️ WORKING MECHANISM:
- Communication shuru hone se pehle Sender aur Receiver ke beech ek **Dedicated Physical Path/Link** 
  banaya jata hai.
- Jab tak baat chal rahi hai (entire conversation), woh rasta book rehta hai aur continuous data flow hota hai.

🛠️ PROS & CONS OF CIRCUIT SWITCHING:
* Advantages (Fayde):
  - Guaranteed Data Rate: Continuous dedicated path milne se fixed bandwidth milti hai.
  - No Waiting Time: Path set hone ke baad intermediate switches par koi waiting delay nahi hota.
  - Continuous Transmission: Badi audio/voice file ya uninterrupted calls ke liye behtar.
* Disadvantages (Nuksan):
  - Dedicated Channel Waste: Agar sender chup bhi baitha hai, tab bhi woh channel koi aur use nahi kar sakta.
  - High Bandwidth Required: Dedicated lines reserve karne mein zyada bandwidth lagti hai.
  - Setup Delay: Physical link establish hone mein shuruat mein kaafi time lagta hai.

-------------------------------------------------------------------------
4. PACKET SWITCHING (Modern Internet Backbone)
-------------------------------------------------------------------------
Modern internet aur shared networks ki efficiency badhane ke liye bade messages ko chhote, 
fixed-sized, numbered "Packets" mein tod diya jata hai. Routers inn packets ko shortest path 
se destination tak pahunchate hain.

🛠️ PROS & CONS OF PACKET SWITCHING:
* Advantages (Fayde):
  - Highly Efficient: Network bandwidth ka optimum aur efficient istemaal hota hai.
  - High Data Transmission: Multiple paths se fast data flow hota hai.
  - Low Memory Need: Chhote packets hone ki wajah se switches mein badi memory ki zaroorat nahi hoti.
  - No Dedicated Path Needed: Dynamic routing se raste apne aap chun liye jaate hain.
  - Best for Voice/Video & Low Cost: Internet/VoIP calling ke liye perfect aur sasta hai.
* Disadvantages (Nuksan):
  - Order Mismatch: Packets alag-alag raste se jaate hain, toh aage-piche (out of order) pahunch sakte hain.
  - Transmission Delay & Packet Loss: Traffic hone par delay ya packet drop hone ka chance rehta hai.
  - Complex Implementation: Routers aur re-assembly logic implement karna complex aur expensive hai.
